User Management

In the admin portal, each user is uniquely identified with a username, password, and profile. Together with other settings, the profile determines which tasks the user can perform, what data the user can see, and what they can do with it. The User Managment section lets the administrator create and manage users, merchant groups, and set roles. You can also view any user session from the Admin Panel. (You can find more information about tracking user sessions here ).


Submenu

User

A user is a person who has an account in the admin portal system. Every user has their own unique identifier that is set at the time the account is created and includes a username, first name, last name, email and their roles.


Get more details


Role

Roles are used to manage permissions to the functions that can be performed in the portal. Roles can be granted permission to perform various functions within all gateways, a particular gateway, with a merchant, or on an account. Roles are basically just a collection of permissions that define the allowed functions, such as Merchant Administrator, etc.


Get more details


Merchant Group

A Merchant Group is a set of merchants who are connected in some way. In the Merchant Groups section, you can create groups which allow users to view the statistics report of the group without having to select each merchant separately.
